# FastAPI App — V0.2 Changes

This document covers everything added on top of the original architecture described in `V(0.1).md`.

---

## New Files

| File | Purpose |
|------|---------|
| `app/models/base.py` | Shared `DeclarativeBase` — replaces the inline `declarative_base()` call that previously lived inside each model |
| `app/models/user.py` | `User` ORM model mapped to the `users` table |
| `app/schemas/users_schema.py` | Pydantic schemas for user input and output (`UserBase`, `UserCreate`, `UserRead`) |
| `app/repositories/user_repository.py` | Database operations for users (`create_user`) |
| `app/services/user_services.py` | Service layer wrapper for user operations |
| `app/api/routes/user_routes.py` | HTTP routes for the users resource |
| `app/utils/security/password_hash.py` | Password hashing and verification via `pwdlib` |
| `app/utils/security/__init__.py` | Exposes `hash_password` and `verify_password` from the `security` package |
| `app/utils/__init__.py` | Package marker for `utils/` |

---

## User Model — `models/user.py`

A new `users` table with the following columns:

| Column | Type | Notes |
|--------|------|-------|
| `id` | UUID | Primary key, generated by `gen_random_uuid()` server-side |
| `name` | String | Non-null, indexed |
| `email` | String | Unique, non-null, indexed |
| `password_hash` | String | Stores the bcrypt hash — never the raw password |
| `created_at` | DateTime (tz-aware) | Set server-side via `now()` |

Uses the modern SQLAlchemy 2.0 `Mapped` / `mapped_column` style (the original `Task` model uses the legacy `Column` style).

---

## Pydantic Schemas — `schemas/users_schema.py`

Three schemas with a clear inheritance chain:

```
UserBase (name, email)
├── UserCreate  — adds password (8–128 chars), used for POST request bodies
└── UserRead    — adds id + created_at, used as the response model
```

`UserRead` sets `model_config = ConfigDict(from_attributes=True)` so it can be built directly from a SQLAlchemy ORM object.

---

## Password Security — `utils/security/password_hash.py`

```python
hash_password(password: str) -> str
verify_password(password: str, hashed_password: str) -> bool
```

Uses `pwdlib` with its recommended algorithm (Argon2/bcrypt). Raw passwords are never stored.

## New Endpoint

| Method | Path | Description |
|--------|------|-------------|
| POST | `/api/users/create` | Register a new user. Returns `UserRead` (no password in response). Responds with `409` if the email is already taken. |

## Changes to Existing Files

### `app/models/task.py`
- Now imports `Base` from `app.models.base` instead of calling `declarative_base()` inline.
- Updated to match the new `tasks` table structure:
  - Added `id` — `BigInteger` primary key (DB-generated via sequence/BIGSERIAL)
  - `task` — no longer the primary key; now `nullable=False`
  - `completed` — now `nullable=False`
  - Added `user_id` — UUID foreign key referencing `users.id`, `nullable=False`

### `app/schemas/task_model.py`
- Replaced the single `Task` schema with three purpose-specific schemas:
  - `TaskCreate` — client input for creating a task (`task: str`, `user_id: uuid.UUID`). No `id` or `completed` — the DB and app set those.
  - `TaskComplete` — client input for completing a task (`id: int` only).
  - `TaskRead` — response shape for all task endpoints (`id`, `task`, `completed`, `user_id`). Has `from_attributes=True` for ORM serialization.

### `app/repositories/task_repository.py`
- `create_task` now accepts `user_id` and passes it to the ORM object.
- `set_complete` now filters by `id` (integer) instead of task name, and raises a `404` if the task is not found.

### `app/services/task_services.py`
- Updated to use `TaskCreate` and `TaskComplete` schemas.
- `create_task` passes `user_id` through to the repository.
- `set_complete` passes `task.id` instead of `task.title`.

### `app/api/routes/task_routes.py`
- Updated imports to use `TaskCreate`, `TaskComplete`, and `TaskRead`.
- Added `response_model=list[TaskRead]` to `GET /api/tasks/get`.
- Added `response_model=TaskRead` to `POST /api/tasks/create`.

### `app/database/session.py`
- `get_db()` now includes an explicit `except Exception: db.rollback(); raise` block before the `finally` close, ensuring transactions are rolled back cleanly on errors.

### `app/main.py`
- Registers the new `user_router` alongside the existing `tasks_router`:
  ```python
  app.include_router(user_router, prefix="/api")
  ```
